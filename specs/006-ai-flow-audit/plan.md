# Implementation Plan: AI Flow System Audit & Fix

**Branch**: `006-ai-flow-audit` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)
**Input**: Full system audit — 61 bugs found across 10 pipeline stages, 5 design decisions confirmed.

## Summary

Fix every bug in the AI pipeline from guest message arrival to AI response delivery. The audit found 61 issues (4 CRITICAL, 9 HIGH, 27 MEDIUM, 21 LOW) across webhooks, debounce, classifier, RAG, topic state, intent extractor, AI service, system prompts, judge, and frontend. Two items already fixed and deployed. This plan prioritizes CRITICAL and HIGH fixes first, then MEDIUM, with LOW deferred.

## Implementation Approach

### Phase 1: CRITICAL + HIGH Fixes (13 bugs)

**Goal**: Eliminate data corruption, double-fires, security violations, and crash risks.

#### Group 1A: Webhook & Message Integrity (AUD-001, AUD-002, AUD-003, AUD-004)

**AUD-001**: Add `@@unique([conversationId, hostawayMessageId])` to Message model. Pre-migration: generate placeholder IDs for existing empty `hostawayMessageId` records. Replace `findFirst` dedup check with `create + P2002 catch` pattern.

**AUD-002 (FR-011)**: In `handleNewMessage()` HOST path (line ~407), call `cancelPendingAiReply(conversation.id, prisma)` when `!isGuest`. Broadcast `ai_typing_clear` SSE event.

**AUD-003**: Add `&& reservation.aiMode !== 'off'` to the scheduling guard at line 403.

**AUD-004**: Change `String(data.conversationId || '')` to `String(data.conversationId ?? '')` (nullish coalescing).

#### Group 1B: Debounce & Double-Fire (AUD-009, AUD-010, AUD-011)

**AUD-009 (FR-014)**: In poll job, replace `markFired(pending.id)` with atomic claim:
```typescript
const claimed = await prisma.pendingAiReply.updateMany({
  where: { id: pending.id, fired: false },
  data: { fired: true },
});
if (claimed.count === 0) continue; // Already claimed by BullMQ
```

**AUD-010**: Await `generateAndSendAiReply` and implement retry on failure (mark `fired: false` to re-queue).

**AUD-011**: Unify aiMode check to whitelist in poll job (same as BullMQ worker).

#### Group 1C: Classifier & RAG Safety (AUD-015, AUD-022, AUD-024, AUD-025)

**AUD-022 + AUD-024**: Eliminate `_lastClassifierResult` module global. Return classifier result directly from `retrieveRelevantKnowledge()` as part of the return object. Pass through to ai.service.ts via the return value.

**AUD-015**: Add dimension validation in `classifyWithLR()`: `if (embedding.length !== coefficients[0].length) throw`.

**AUD-025 (FR-013)**: Add `propertyAmenities` parameter to `retrieveRelevantKnowledge()`. Pass to `getSopContent()` in HIGH and MEDIUM paths.

#### Group 1D: Topic State & Judge (AUD-029, AUD-030, AUD-052)

**AUD-029 (FR-015)**: Replace keyword substring matching with centroid-only detection. Move keyword check AFTER centroid check as fallback when centroids unavailable.

**AUD-030 (FR-009)**: Run topic switch detection independently of Tier 1 confidence. In ai.service.ts, check centroid distance even when `ragResult.tier === 'tier1'` — compare new classification against cached topic.

**AUD-052**: Change judge.service.ts line 14 from top-level `new Anthropic(...)` to lazy `getClient()` pattern (same as intent-extractor.service.ts).

### Phase 2: MEDIUM Fixes (27 bugs)

**Goal**: Improve AI response quality, debugging, and operational reliability.

#### Group 2A: AI Service Core (AUD-037, AUD-038, AUD-039, AUD-040, AUD-041)

**AUD-037 (FR-001)**: After all tiers complete (RAG + Tier 3 + Tier 2), deduplicate `retrievedChunks` by `category` before building the prompt.

**AUD-038**: Check `ragQuery.trim().length > 0` before calling RAG. For image-only messages, use image description or skip RAG.

**AUD-039**: Extract escalation validation into shared helper `validateEscalation(parsed)`. Call from both text and image branches.

**AUD-040 (FR-012)**: After detecting escalation signals, inject into content blocks:
```typescript
if (escalationSignals.length > 0) {
  propertyInfo += '\n### SYSTEM SIGNALS\n' +
    escalationSignals.map(s => `⚠ ${s.signal}: ${s.description}`).join('\n');
}
```

**AUD-041**: Reduce `WEBHOOK_DELIVERY_BUFFER_MS` from 30 minutes to 10 minutes.

#### Group 2B: RAG Pipeline (AUD-023, AUD-026, AUD-027, AUD-028)

**AUD-023 + AUD-028**: Add `intentExtractorRan: boolean` to RAG return type. Set `true` when LOW path fires intent extractor. ai.service.ts skips retry when flag is set.

**AUD-026**: Return `confidenceTier: 'low'` from pgvector fallback path. Set `isFallback: true` flag.

**AUD-027**: Use separate threshold for pgvector fallback (0.5 cosine sim instead of 0.85 LR confidence).

#### Group 2C: System Prompts (AUD-044, AUD-045, AUD-046)

**AUD-044 (FR-003)**: Add to `ai-config.json` system prompt:
```
IMPORTANT: When an SOP mentions date-based conditions (e.g., "within 2 days of check-in"),
ALWAYS compare against the check-in date in RESERVATION DETAILS and the current local time
to determine which branch applies. Never use the generic response when the date condition is met.
```

**AUD-045**: Remove dead code `OMAR_SYSTEM_PROMPT` and `OMAR_SCREENING_SYSTEM_PROMPT` constants. Keep only the runtime `ai-config.json` versions.

**AUD-046**: Replace fragile `string.replace()` in `injectImageHandling` with template variable `{IMAGE_HANDLING}`.

#### Group 2D: Judge & Self-Improvement (AUD-053, AUD-054, AUD-055)

**AUD-053 + AUD-054**: Add `if (validLabels.length > 0)` guard before every `addExample()` call.

**AUD-055**: Global debounce on `reinitializeClassifier()` — coalesce calls within a 60-second window.

#### Group 2E: Frontend & Observability (AUD-057, AUD-058, AUD-059)

**AUD-057 (FR-007)**: Remove `.substring(0, 200)` from ragContext chunk content storage. Store full text.

**AUD-058 (FR-008)**: Add `llmOverride` to pipeline feed response.

**AUD-059 (FR-005/006)**: Add `centroidSimilarity`, `centroidThreshold`, `switchMethod` to `getReinjectedLabels()` return, ragContext, pipeline feed, and frontend display.

#### Group 2F: Other MEDIUM (AUD-005, AUD-006, AUD-012, AUD-013, AUD-016, AUD-017, AUD-031, AUD-033)

**AUD-005**: Update guest name during resync when it differs from stored value.
**AUD-006**: Validate parsed date; fall back to `new Date()` if invalid.
**AUD-012**: Use timezone-aware arithmetic for DST in `nextWorkingHoursStart`.
**AUD-013**: Use `Intl.DateTimeFormat` for reliable midnight computation.
**AUD-016**: Add dimension mismatch guard to `cosineSimilarity()`.
**AUD-017**: Call `loadLrWeightsMetadata()` at end of `initializeClassifier()`.
**AUD-031**: Add LRU cap to topic state cache (max 10,000 entries).
**AUD-033**: Add semaphore to intent extractor (max 10 concurrent Tier 2 calls).
