# Implementation Plan: Inquiry Accept/Reject

**Branch**: `029-inquiry-accept-reject` | **Date**: 2026-04-03 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/029-inquiry-accept-reject/spec.md`

## Summary

Add the ability for property managers to approve, reject, and cancel reservation inquiries directly from GuestPilot. This requires a "Connect Hostaway Dashboard" flow in settings that extracts a 90-day session token via bookmarklet, and new action buttons in the inbox list and conversation right panel that call Hostaway's internal dashboard API (`platform.hostaway.com`).

## Technical Context

**Language/Version**: TypeScript 5.x on Node.js 18+ (backend), Next.js 16 + React 19 (frontend)  
**Primary Dependencies**: Express 4.x, Prisma ORM, axios, Tailwind 4, shadcn/ui  
**Storage**: PostgreSQL + Prisma ORM (new fields on Tenant, new InquiryActionLog model)  
**Testing**: Manual testing against live Hostaway dashboard API  
**Target Platform**: Railway (backend), Vercel (frontend)  
**Project Type**: Web application (backend API + frontend SPA)  
**Performance Goals**: <5s for approve/reject action round-trip (SC-001)  
**Constraints**: Must not break core messaging flow (Constitution §I). Token encrypted at rest (FR-015).  
**Scale/Scope**: Multi-tenant, one dashboard connection per tenant

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Notes |
|-----------|--------|-------|
| §I Graceful Degradation | PASS | Dashboard connection is optional. If not connected/expired, actions blocked with clear UI. Core messaging unaffected. All Hostaway dashboard API calls are independent of the guest messaging pipeline. |
| §II Multi-Tenant Isolation | PASS | Dashboard JWT stored per-tenant. All action endpoints require auth middleware with tenantId. InquiryActionLog scoped by tenantId. |
| §III Guest Safety | PASS | Feature is manager-facing only. No guest-visible changes. No access code exposure risk. |
| §IV Structured AI Output | N/A | No AI involvement in this feature. |
| §V Escalate When In Doubt | N/A | No AI decision-making involved. |
| §VI Observability | PASS | InquiryActionLog records every action with outcome. Console logging for Hostaway API calls follows existing pattern. |
| §VII Tool-Based Architecture | N/A | Not an AI tool — direct manager action. |
| §VIII FAQ Knowledge Loop | N/A | Not related. |
| Security | PASS | Dashboard JWT encrypted via AES-256-GCM. No credentials stored. Token never logged or exposed in API responses. |
