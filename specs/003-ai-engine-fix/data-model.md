## New Artifacts (File-Based, Not DB)

### ClassifierWeights (JSON file)

Stored at: `backend/src/config/classifier-weights.json`
Generated by: Python training script
Loaded at: startup + on retrain (atomic swap)

```json
{
  "model": "cohere-embed-multilingual-v4.0",
  "trainedAt": "2026-03-19T12:00:00Z",
  "exampleCount": 450,
  "classes": ["sop-cleaning", "sop-amenity-request", ...],
  "coefficients": [[0.123, -0.456, ...], ...],
  "intercepts": [0.789, ...],
  "centroids": {
    "sop-cleaning": [0.111, 0.222, ...],
    "sop-amenity-request": [0.333, 0.444, ...]
  },
  "thresholds": {
    "global": 0.65,
    "perCategory": {
      "sop-cleaning": 0.58,
      "sop-amenity-request": 0.62
    }
  },
  "calibration": {
    "crossValAccuracy": 0.87,
    "perCategoryAccuracy": {
      "sop-cleaning": 0.92,
      "sop-amenity-request": 0.85
    }
  }
}
```

**Fields**:
- `classes`: ordered category names matching coefficient rows
- `coefficients`: [n_classes x embedding_dim] weight matrix
- `intercepts`: [n_classes] bias terms
- `centroids`: mean embedding per category (for topic switch)
- `thresholds.global`: overall rejection threshold (calibrated)
- `thresholds.perCategory`: per-category thresholds (mean - 2*std)
- `calibration`: cross-validation accuracy results

## Three-Tier Routing State

The RAG service needs to know which tier to use. This is derived
from the LR sigmoid confidence at classification time:

```typescript
type ConfidenceTier = 'high' | 'medium' | 'low';

interface ClassificationResult {
  labels: string[];                // all labels above per-label sigmoid threshold
  confidence: number;              // max sigmoid score across all returned labels
  tier: ConfidenceTier;
  topCandidates: Array<{ label: string; confidence: number }>;  // top 3 by individual per-label sigmoid score
  knnDiagnostic: {  // kept for pipeline display
    topSimilarity: number;
    neighbors: Array<{ text: string; labels: string[]; similarity: number }>;
    method: string;
  };
}
```

The tier determines SOP injection strategy:
- `high`: inject single SOP for `labels[0]`
- `medium`: inject SOPs for `topCandidates[0..2]` + verification prompt
- `low`: fire intent extractor → inject matched SOPs

## Schema Changes (DB)

No database schema changes needed. The LR weights are file-based
(loaded into memory at startup). Training examples stay in
ClassifierExample table (existing). Centroids and thresholds are
derived from training data during the retrain step.

Both Railway services (`backend-advanced-ai` and `backend-new-ai`)
share the same database. The `ragContext` JSON field in AiApiLog
stores engine-specific data — the frontend handles both formats.

## API Changes

### POST /api/knowledge/retrain-classifier (NEW)

Triggers LR retraining:
1. Fetches all active training examples from DB
2. Embeds with Cohere (input_type="classification")
3. Runs Python child process (sklearn LR training)
4. Writes new classifier-weights.json
5. Triggers atomic swap of ClassifierState
6. Returns training results (accuracy, example count, duration)

### GET /api/knowledge/classifier-status (MODIFIED)

Add to response:
- `classifierType`: "lr" (was "knn")
- `lrAccuracy`: cross-validation accuracy
- `lastTrainedAt`: timestamp
- `retrainAvailable`: boolean (true if examples changed since last train)
