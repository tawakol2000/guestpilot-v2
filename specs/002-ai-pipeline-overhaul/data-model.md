# Data Model Changes: AI Pipeline Overhaul

**Date**: 2026-03-19
**Feature**: 002-ai-pipeline-overhaul

## Schema Changes

### TenantAiConfig — Judge Mode Toggle

**Change**: Add `judgeMode` field for manual judge behavior control.

```prisma
judgeMode String @default("evaluate_all")
```

**Values**:
- `"evaluate_all"` — Judge fires on every AI response (default while
  training set is small)
- `"sampling"` — Judge applies existing skip conditions + 30% random
  sampling of confident messages

**Validation**: MUST be one of the two allowed values.

---

### ClassifierEvaluation — Skip Reason Logging

**Change**: Add `skipReason` field to log why the judge didn't evaluate
a message (when it does skip).

```prisma
skipReason String? @default(null)
```

**Values** (when populated):
- `"judge_mode_sampling_skip"` — Random sample excluded this message
- `"confident_skip"` — topSim >= threshold (only in sampling mode)
- `"tier3_skip"` — Tier 3 re-injected (only in sampling mode)
- `null` — Judge evaluated (no skip)

**Note**: In `evaluate_all` mode, skipReason is never set because
the judge always fires.

---

### ClassifierExample — Gap Analysis Source

**Change**: No schema change needed. The `source` field already
accepts any string. New sources to use:

- `"gap-analysis"` — Generated by the gap analysis tool, pending
  operator approval (stored with `active: false` until approved)
- `"operator-correction"` — From thumbs-down rating with correction
  (future US6)

**Approval flow**: Gap-analysis examples are created with
`active: false`. When operator approves, set `active: true`. When
operator rejects, delete the record.

---

## No New Models

All changes are additions to existing models. No new database tables
are introduced.

## API Endpoints (New)

### GET /api/ai-pipeline/accuracy

Returns aggregated accuracy metrics for the pipeline dashboard.

**Query params**: `period` (7d or 30d, default 30d)

**Response**:
```json
{
  "overall": { "correct": 40, "total": 50, "accuracy": 0.80 },
  "emptyLabelRate": 0.08,
  "perCategory": [
    { "category": "sop-maintenance", "correct": 12, "total": 14, "accuracy": 0.857 }
  ],
  "selfImprovement": {
    "totalActive": 87,
    "bySource": { "manual": 10, "llm-judge": 42, "tier2-feedback": 30, "gap-analysis": 5 },
    "addedThisPeriod": 23
  },
  "judgeMode": "evaluate_all"
}
```

### POST /api/knowledge/gap-analysis

Runs gap analysis and generates suggested training examples.

**Response**:
```json
{
  "emptyLabelMessages": 17,
  "underrepresentedCategories": [
    { "category": "sop-long-term-rental", "count": 5 }
  ],
  "languageDistribution": { "ar": 42, "en": 28, "other": 3 },
  "suggestedExamples": 25,
  "message": "Generated 25 suggested examples. Review in classifier examples UI."
}
```

### POST /api/ai-pipeline/snapshot

Generates a pipeline snapshot and writes to
`.specify/memory/pipeline-snapshot.md`.

**Response**: The markdown content as `text/markdown`.

### POST /api/knowledge/classifier-examples/:id/approve

Approves a suggested example (sets `active: true`).

### POST /api/knowledge/classifier-examples/:id/reject

Rejects a suggested example (deletes the record).

### POST /api/knowledge/batch-classify

Batch classification for threshold tuning.

**Body**: `{ messages: string[], voteThreshold?: number }`

**Response**: Classification results for each message at the given
threshold.
